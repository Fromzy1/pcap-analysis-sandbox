# Eval harness

How the [pcap-analysis skill](../pcap-analysis/) was measured. This is the
evidence behind the benchmark in [`examples/`](../../examples/before-after-benchmark.md);
it is dev-time tooling, not part of the shipped skill.

## What's here

| Path | What it is |
|------|------------|
| `grade.py` | Deterministic grader — scores each run's `analysis.md` against fixed assertions. **No LLM in the loop.** |
| `iteration-1/` | Graded run outputs: per scenario, `with_skill/` and `without_skill/` reports, plus `grading.json`. |
| `iteration-1-review.html` | Self-contained HTML viewer for the runs (pass/fail per assertion, side by side). |
| `benchmark.md`, `benchmark.json` | Aggregated results across runs. |

The synthetic test captures themselves are generated by
[`../pcap-analysis/evals/make_test_pcaps.py`](../pcap-analysis/evals/make_test_pcaps.py)
and the scenarios/assertions are defined in
[`../pcap-analysis/evals/evals.json`](../pcap-analysis/evals/evals.json).

## How grading works

`grade.py` reads each scenario's `eval_metadata.json` (the assertion list), then
runs a Python predicate per assertion against the report text — e.g.
`mentions_zero_window`, `names_failing_domains`, `cites_frame_numbers` (passes at
≥3 unique frames), `cites_specific_filter` (looks for real `tshark`/display-filter
syntax). Predicates are plain regex/heuristics, so grading is **objective, fast,
and free of grader drift**.

The sharpest one is `example_ip_falsely_flagged`: it scans a ±2-line window around
the benign decoy IP (`93.184.216.34` / example.com) and fails the run only if the
IP is *actively* labelled malicious ("secondary C2", "blocklist", "indicator of
compromise") **without** a nearby rescue phrase ("benign", "connectivity canary",
"not malicious"). That's how the harness catches the baseline's false positive
without penalizing a report that mentions the IP just to clear it.

## Run it

```sh
# from the repo root, with the sandbox active (see ../../SETUP.md)
cd skill-dev/pcap-analysis-workspace
python grade.py          # writes grading.json next to each run, prints pass counts
```
