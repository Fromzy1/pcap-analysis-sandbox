# Examples — what the pcap-analysis skill does

Three worked examples, each a real question someone might ask about a packet
capture, the sample `.pcap` it was asked about, and the **actual report the skill
produced**. They cover the skill's three angles: security/IR, performance RCA, and
protocol/DNS root-cause. Every capture lives in [`../pcaps/`](../pcaps/) and is
small enough to open in Wireshark or re-run yourself.

| Example | Angle | The question | What it demonstrates |
|---------|-------|--------------|----------------------|
| [**beacon-c2-detection**](beacon-c2-detection/) | Security / IR | "SOC flagged this workstation — anything concerning? I need IOCs." | Beacon periodicity detection, a SIEM-ready IOC table, and **not** false-flagging a benign host |
| [**dns-cascade-502**](dns-cascade-502/) | Performance / RCA | "We're getting 502s — DNS, backend, or both?" | Separating a DNS failure (NXDOMAIN) from an application failure (HTTP 502) |
| [**slow-checkout-stall**](slow-checkout-stall/) | Performance / RCA | "Checkout hangs for seconds — what's wrong?" | Pinning a stall to a TCP zero-window event with the exact frame + filter |

Each folder has a short `README.md` (the question, the headline result, and a
copy-paste reproduce block) and an `analysis.md` (the full report — the deliverable).

## Start here

- **Most impressive:** [beacon-c2-detection](beacon-c2-detection/) — the hardest
  scenario, where disciplined analysis avoids a costly false-positive IOC.
- **Shortest read:** [slow-checkout-stall](slow-checkout-stall/) — one capture,
  one clear root cause.
- **Does it actually help?** [before-after-benchmark.md](before-after-benchmark.md) —
  with-skill **100%** vs without-skill **80%** pass rate on a deterministic grader,
  and the false-positive the baseline produced.

## Running an example yourself

From the repo root:

```sh
source ./activate.sh                                       # enter the sandbox env (tshark, scapy, …)
bash skill-dev/pcap-analysis/scripts/triage.sh pcaps/eval-beacon.pcap
# then ask Claude (with the pcap-analysis skill loaded) the question from the example's README
```

The captures are synthetic (generated by
[`skill-dev/pcap-analysis/evals/make_test_pcaps.py`](../skill-dev/pcap-analysis/evals/make_test_pcaps.py))
so they're safe to share and each embeds exactly the signal the investigation
should find.
